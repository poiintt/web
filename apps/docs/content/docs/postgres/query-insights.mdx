Query Insights gives Prisma Postgres developers instant visibility into what is slow and why by connecting Prisma code to the SQL it generates, with no setup required.

:::note
Query Insights is the evolution of [Prisma Optimize](/optimize). It is now built directly into Prisma Postgres with no additional setup or extensions needed.
:::

## What you can do

- **Identify slow queries and performance bottlenecks** — see execution time for grouped queries
- **Track reads per query group** — monitor which queries consume the most resources
- **ORM-to-SQL attribution** — see which Prisma query in your code generated the SQL (exclusive to Prisma ORM)
- **AI-powered optimization** — generate a prompt to refine and optimize queries, such as adding indexes
- **Monitor query history** — track when a query was last executed

## Why use Query Insights

### Save cost

- Reduce dependency on third-party monitoring tools
- Catch inefficient queries early to lower code maintenance
- Improve query performance to reduce compute and database costs

### Improve performance

- Detect slow queries before they become production incidents
- Identify high-read query patterns that impact scalability
- Continuously refine queries to keep applications responsive

## How it works

Query Insights works out of the box with Prisma Postgres — no agents, extensions, or instrumentation required.

1. Open Query Insights in your Prisma Postgres dashboard
2. Analyze query groups to understand execution time, read volume, and usage patterns
3. Identify inefficient or frequently executed queries
4. Use the generated AI prompt in your editor to improve the query (e.g., adding indexes, fixing N+1 patterns)
5. Monitor improvements over time

## ORM-aware insights

SQL-only tools show expensive queries. Query Insights shows which Prisma query in your code generated them.

This enables:

- Attribution from endpoint or code path to Prisma query to SQL impact
- Understanding changes at the application layer, not just database metrics
- Fixes that map directly to developer actions such as adjusting query shape, relation loading, or indexing

:::note
ORM-to-SQL attribution works with Prisma ORM. Query Insights also works for SQL queries from other ORMs, but without the ORM-level attribution.
:::

## Included with Prisma Postgres

Query Insights is included with Prisma Postgres at no additional cost. No separate setup, API keys, or extensions are needed.
