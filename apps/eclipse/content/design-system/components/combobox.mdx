### Usage

**Basic Combobox**

A simple combobox with searchable options:

```tsx
import {
  Combobox,
  ComboboxInput,
  ComboboxContent,
  ComboboxList,
  ComboboxItem,
} from "@prisma/eclipse";

const fruits = ["Apple", "Banana", "Orange", "Grape", "Mango"];

export function BasicCombobox() {
  return (
    <Combobox>
      <ComboboxInput placeholder="Select a fruit..." />
      <ComboboxContent>
        <ComboboxList>
          {fruits.map((fruit) => (
            <ComboboxItem key={fruit} value={fruit}>
              {fruit}
            </ComboboxItem>
          ))}
        </ComboboxList>
      </ComboboxContent>
    </Combobox>
  );
}
```

**Multi-Select with Chips**

Enable multiple selections with removable chips:

```tsx
import {
  Combobox,
  ComboboxChips,
  ComboboxChip,
  ComboboxChipsInput,
  ComboboxContent,
  ComboboxList,
  ComboboxItem,
  useComboboxAnchor,
} from "@prisma/eclipse";

const tags = ["React", "TypeScript", "Next.js", "Tailwind", "Prisma"];

export function MultiSelectCombobox() {
  const [selectedTags, setSelectedTags] = React.useState<string[]>([]);
  const anchor = useComboboxAnchor();

  return (
    <Combobox multiple value={selectedTags} onValueChange={setSelectedTags}>
      <ComboboxChips ref={anchor}>
        {selectedTags.map((tag) => (
          <ComboboxChip key={tag} value={tag}>
            {tag}
          </ComboboxChip>
        ))}
        <ComboboxChipsInput placeholder="Add tags..." />
      </ComboboxChips>
      <ComboboxContent anchor={anchor.current}>
        <ComboboxList>
          {tags.map((tag) => (
            <ComboboxItem key={tag} value={tag}>
              {tag}
            </ComboboxItem>
          ))}
        </ComboboxList>
      </ComboboxContent>
    </Combobox>
  );
}
```

**With Custom Filtering**

Implement custom filtering logic for more control:

```tsx
import {
  Combobox,
  ComboboxInput,
  ComboboxContent,
  ComboboxList,
  ComboboxItem,
  ComboboxEmpty,
} from "@prisma/eclipse";

const users = [
  { id: "1", name: "John Doe", email: "john@example.com" },
  { id: "2", name: "Jane Smith", email: "jane@example.com" },
  { id: "3", name: "Bob Johnson", email: "bob@example.com" },
];

export function CustomFilterCombobox() {
  const [searchValue, setSearchValue] = React.useState("");
  
  const filteredUsers = users.filter((user) => {
    const search = searchValue.toLowerCase();
    return (
      user.name.toLowerCase().includes(search) ||
      user.email.toLowerCase().includes(search)
    );
  });

  return (
    <Combobox>
      <ComboboxInput
        placeholder="Search users..."
        value={searchValue}
        onValueChange={setSearchValue}
      />
      <ComboboxContent>
        <ComboboxList>
          {filteredUsers.map((user) => (
            <ComboboxItem key={user.id} value={user.id}>
              <div className="flex flex-col items-start">
                <span>{user.name}</span>
                <span className="text-xs text-muted-foreground">
                  {user.email}
                </span>
              </div>
            </ComboboxItem>
          ))}
        </ComboboxList>
        <ComboboxEmpty>No users found.</ComboboxEmpty>
      </ComboboxContent>
    </Combobox>
  );
}
```

### API Reference

#### Combobox

The root component that manages combobox state.

**Props:**
- `value` - Controlled value (string or string[], optional)
- `onValueChange` - Callback when value changes ((value: string | string[]) => void, optional)
- `open` - Controlled open state (boolean, optional)
- `onOpenChange` - Callback when open state changes ((open: boolean) => void, optional)
- `multiple` - Enable multi-select mode (boolean, default: false)
- `disabled` - Disable the combobox (boolean, default: false)
- `defaultValue` - Default value for uncontrolled mode (string or string[], optional)
- `defaultOpen` - Default open state (boolean, default: false)

#### ComboboxInput

The input field that triggers the combobox and filters options.

**Props:**
- `placeholder` - Placeholder text (string, optional)
- `disabled` - Disable the input (boolean, default: false)
- `showTrigger` - Show the dropdown trigger button (boolean, default: true)
- `showClear` - Show the clear button (boolean, default: false)
- `className` - Additional CSS classes (string, optional)
- All standard HTML input attributes

#### ComboboxContent

The dropdown container that holds the list of options.

**Props:**
- `side` - Position relative to trigger ("top" | "bottom" | "left" | "right", default: "bottom")
- `align` - Alignment relative to trigger ("start" | "center" | "end", default: "start")
- `sideOffset` - Distance from trigger in pixels (number, default: 6)
- `alignOffset` - Alignment offset in pixels (number, default: 0)
- `anchor` - Custom anchor element for positioning (HTMLElement, optional)
- `className` - Additional CSS classes (string or function, optional)

#### ComboboxList

The scrollable list container for combobox items.

**Props:**
- `className` - Additional CSS classes (string or function, optional)
- All standard HTML div attributes

#### ComboboxItem

An individual option in the combobox list.

**Props:**
- `value` - The value of the item (string, required)
- `disabled` - Disable the item (boolean, default: false)
- `className` - Additional CSS classes (string or function, optional)
- `children` - Item content (ReactNode)

#### ComboboxGroup

A container for grouping related items.

**Props:**
- `className` - Additional CSS classes (string or function, optional)
- `children` - Group content (ReactNode)

#### ComboboxLabel

A label for a group of items.

**Props:**
- `className` - Additional CSS classes (string or function, optional)
- `children` - Label content (ReactNode)

#### ComboboxEmpty

Displays when no items match the search.

**Props:**
- `className` - Additional CSS classes (string or function, optional)
- `children` - Empty state content (ReactNode)

#### ComboboxSeparator

A visual separator between groups or items.

**Props:**
- `className` - Additional CSS classes (string or function, optional)

#### ComboboxChips

Container for multi-select chips (used with multiple mode).

**Props:**
- `className` - Additional CSS classes (string or function, optional)
- `children` - Chips and input (ReactNode)

#### ComboboxChip

An individual chip representing a selected item.

**Props:**
- `value` - The value of the chip (string, required)
- `showRemove` - Show the remove button (boolean, default: true)
- `className` - Additional CSS classes (string or function, optional)
- `children` - Chip content (ReactNode)

#### ComboboxChipsInput

Input field within the chips container for multi-select.

**Props:**
- `placeholder` - Placeholder text (string, optional)
- `className` - Additional CSS classes (string or function, optional)
- All standard HTML input attributes

#### ComboboxValue

Displays the currently selected value.

**Props:**
- `placeholder` - Text to show when no value is selected (string, optional)

#### ComboboxTrigger

A custom trigger button for opening the combobox dropdown. Automatically includes a chevron icon.

**Props:**
- `className` - Additional CSS classes (string or function, optional)
- `children` - Custom trigger content (ReactNode, optional)
- `disabled` - Disable the trigger (boolean, default: false)
- All standard HTML button attributes

#### ComboboxCollection

A component for managing dynamic collections of items. Useful when working with data that changes frequently.

**Props:**
- `items` - Array of items to render (array, required)
- `children` - Render function for items (ReactNode)

#### useComboboxAnchor

Hook that returns a ref for custom positioning with chips.

**Returns:** `React.RefObject<HTMLDivElement | null>`

- Always provide a `placeholder` to guide users
- Use `ComboboxEmpty` to inform users when no results are found
- Group related options with `ComboboxGroup` and `ComboboxLabel`
- For large lists, implement virtual scrolling or pagination
- Provide clear visual feedback for selected items

- Consider debouncing search input for async data fetching
- Test keyboard navigation thoroughly
- Provide appropriate ARIA labels for screen readers
- Use the clear button for better user experience in forms

### Accessibility

The Combobox component follows ARIA combobox pattern specifications:

- Uses semantic ARIA roles (`combobox`, `listbox`, `option`)
- Supports keyboard navigation:
  - `Arrow Down/Up` - Navigate through options
  - `Enter` - Select highlighted option
  - `Escape` - Close the dropdown
  - `Tab` - Move focus and close dropdown
- Focus management and trapped focus in dropdown
- Screen reader announcements for selections and state changes
- Proper labeling with `aria-label` and `aria-labelledby`
- Visual focus indicators
- Selected state announced to assistive technologies
- Disabled items are properly marked and unfocusable

### Performance Tips

For large datasets:

1. **Implement pagination** - Load items as user scrolls
2. **Debounce search input** - Reduce filter calculations
3. **Use React.memo** - Memoize item components
4. **Virtual scrolling** - Render only visible items
5. **Async loading** - Fetch data on-demand

```tsx
import { useMemo, useState } from "react";
import { useDebounce } from "@/hooks/use-debounce";

export function OptimizedCombobox({ items }) {
  const [search, setSearch] = useState("");
  const debouncedSearch = useDebounce(search, 300);
  
  const filteredItems = useMemo(
    () => items.filter(item => 
      item.toLowerCase().includes(debouncedSearch.toLowerCase())
    ),
    [items, debouncedSearch]
  );

  return (
    <Combobox>
      <ComboboxInput 
        value={search}
        onValueChange={setSearch}
        placeholder="Search..."
      />
      <ComboboxContent>
        <ComboboxList>
          {filteredItems.map((item) => (
            <ComboboxItem key={item} value={item}>
              {item}
            </ComboboxItem>
          ))}
        </ComboboxList>
      </ComboboxContent>
    </Combobox>
  );
}
```
